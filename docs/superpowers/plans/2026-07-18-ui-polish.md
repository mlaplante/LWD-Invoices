# UI Polish Pass Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the consistency and accessibility gaps found in the 2026-07-18 UI audit — zero behavior/data-flow changes, purely presentational/a11y/copy fixes.

**Architecture:** Six independent tasks. The stack: Tailwind v4 CSS-first tokens (`src/app/globals.css`, `@theme inline`, no tailwind.config), shadcn/Radix primitives in `src/components/ui/`, sonner toasts, shared formatters in `src/lib/format.ts` (`formatCurrency(n, symbol, symbolPosition, code)` at :87, `formatDate` et al).

**Audited facts (2026-07-18 — trust, don't re-derive):**
- `/reconciliation` and `/replies` are MISSING from `src/components/layout/MobileNav.tsx:46-64` `moreItems` — unreachable on mobile.
- `src/components/replies/ReplyTriageList.tsx`: bare `<p>No replies to triage</p>` empty state (line ~8), NO loading state (`rows.isLoading` never read), NO toast on the `recategorize` mutation. The dominant empty-state pattern is a dashed-border card (see `ExpenseList.tsx` / `ProposalList.tsx`).
- `src/components/reconciliation/UnmatchedPaymentsList.tsx:10` + `MatchPaymentDialog.tsx` use a local `money()` helper hardcoding USD/en-US instead of `formatCurrency` from `src/lib/format.ts:87`; `UnmatchedPaymentsList.tsx:28` uses raw `toLocaleDateString()` instead of `formatDate`, and hand-rolls a status pill instead of `<Badge variant="secondary">`.
- The four new files (`reconciliation/page.tsx`, `replies/page.tsx`, `UnmatchedPaymentsList.tsx`, `ReplyTriageList.tsx`) are single-line unformatted JSX blobs — need Prettier formatting.
- Route-level `loading.tsx` with `Skeleton` is the dominant loading pattern (39 files, e.g. `src/app/(dashboard)/invoices/loading.tsx`); reconciliation/replies lack one.
- `src/components/ui/dialog.tsx:64` `DialogContent` has `max-w-[calc(100%-2rem)]` but NO max-height/scroll — 24 of 28 usages can overflow short viewports.
- `ClientForm.tsx` (~lines 159-243), `InvoiceForm.tsx` (513, 537, 556, 604 …), `ExpenseForm.tsx` (215-396) use bare `<label>` with no `htmlFor`/`id` (exceptions: `InvoiceForm.tsx:403`, `ExpenseForm.tsx:387`). `ClientForm.tsx:198` has `aria-invalid` but no `aria-describedby` to the error at 204-209.
- `AddUnmatchedPaymentDialog.tsx`: all text fields correctly labeled; the Method `<Select>` label (line ~58) is unassociated; the submit error `<p>` (line ~62) lacks `aria-describedby`/live semantics.
- `MatchPaymentDialog.tsx:51`: over-allocation state is color-only (`text-destructive` toggle).
- Chat surfaces: `ChatAssistant.tsx` has an icon-only send button with no `aria-label` (117-123) and no `aria-live` region for streamed messages; `ReportsAskPanel.tsx` has `aria-label` on send (87) but no live region either. `InvoiceForm.tsx:423,442` shows the house `role="status" aria-live="polite"` pattern.
- Focus visibility: global rule at `globals.css:136-137`; three inputs opt out via bare `outline-none`: `ChatAssistant.tsx:114`, `ReportsAskPanel.tsx:86`, `CommandPalette.tsx:120`.
- No `prefers-reduced-motion` handling anywhere; `tw-animate-css` drives dialog/sheet animations.
- `ReportsAskPanel.tsx:68,77,87` hand-rolls `<button>`s instead of the `Button` primitive (`ui/button.tsx` has ghost/icon variants).
- Sidebar `secondaryNav` section is labeled "Analytics" (`SidebarNav.tsx:105-107`) but holds 12 mixed items (ops + settings + analytics).
- Copy drift: `invoices/page.tsx:109` "+ New Invoice" vs `:158` "Create Invoice"; `clients/page.tsx:86` "+ New Client" vs `:134` "Add Client"; `projects/page.tsx:89,128` "+ New Project" vs "Create Project".
